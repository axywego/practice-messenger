# TCP Мессенджер на Boost.Asio

## 1. Описание проекта

Реализовать клиент-серверный мессенджер на C++ с использованием Boost.Asio для асинхронного
сетевого взаимодействия. Приложение поддерживает регистрацию и аутентификацию пользователей,
обмен сообщениями в общем чате, систему друзей и личные сообщения между ними,
а также передачу файлов с шифрованием на стороне сервера.

## 2. Цель проекта

Реализовать мессенджер, который позволяет:

- Регистрироваться и входить в систему с хешированием паролей
- Общаться в общем чате в реальном времени
- Добавлять пользователей в друзья и обмениваться личными сообщениями
- Получать личные сообщения, отправленные пока пользователь был офлайн
- Загружать и скачивать файлы с шифрованием AES-256

## 3. Компоненты системы

### Сервер

**Функции:**

- Регистрация и аутентификация пользователей (выдача сессионных токенов)
- Хранение данных в базе данных: пользователи, дружба, история сообщений
- Маршрутизация сообщений: broadcast для общего чата, адресная доставка для личных
- Офлайн-доставка: личные сообщения хранятся в БД и отправляются при следующем входе
- Управление системой друзей: заявки, принятие, отклонение

### Клиент

**Функции:**

- Регистрация и вход в систему
- Отправка и получение сообщений общего чата в реальном времени
- Отправка и получение личных сообщений (только между друзьями)
- Управление списком друзей
- Загрузка и скачивание файлов

## 5. Архитектура протокола

Каждый пакет состоит из заголовка фиксированного размера и тела переменной длины:

```
[4 байта: PacketType в big-endian][4 байта: длина тела в big-endian][тело]
```

Типы пакетов:

| Тип               | Значение | Направление        | Описание                        |
|-------------------|----------|--------------------|---------------------------------|
| REGISTER          | 1        | Client → Server    | Регистрация                     |
| LOGIN             | 2        | Client → Server    | Вход                            |
| UPLOAD            | 3        | Client → Server    | Загрузка файла                  |
| DOWNLOAD          | 4        | Client → Server    | Скачивание файла                |
| MESSAGE           | 5        | оба направления    | Общий чат                       |
| RESULT            | 6        | Server → Client    | Универсальный ответ             |
...

## 6. Схема базы данных

```sql
CREATE TABLE users(
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    login TEXT UNIQUE NOT NULL,
    salt TEXT NOT NULL,
    password TEXT NOT NULL
);

-- user1 < user2 всегда
CREATE TABLE friendships(
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user1 INTEGER NOT NULL,
    user2 INTEGER NOT NULL,
    status TEXT NOT NULL DEFAULT 'pending',
    initiator INTEGER NOT NULL,
    FOREIGN KEY(user1) REFERENCES users(id),
    FOREIGN KEY(user2) REFERENCES users(id)
);

CREATE TABLE direct_messages(
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    sender INTEGER NOT NULL,
    recipient INTEGER NOT NULL,
    create_at INTEGER NOT NULL,
    delivered INTEGER NOT NULL,
    FOREIGN KEY(sender) REFERENCES users(id),
    FOREIGN KEY(recipient) REFERENCES users(id)
);

CREATE TABLE sessions (
    token TEXT PRIMARY KEY,
    id INTEGER NOT NULL,
    created_at INTEGER NOT NULL,
    FOREIGN KEY (id) REFERENCES users(id)
);
```
